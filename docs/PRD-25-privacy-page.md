# PRD-25：隐私策略页面

> **目标**：为 BrickPlans 添加隐私策略页面，满足合规要求。
> **关联**：docs/privacy-policy.md（内容已撰写）

---

## 一、需求概述

新增 `#/privacy` 路由，渲染隐私策略内容。页面底部加「隐私策略」入口链接。

---

## 二、前端实现

### 2.1 路由注册

在 `main.js` 路由分发处新增 `case 'privacy'`，渲染 `renderPrivacyPage()`。

### 2.2 页面结构

```
┌─ 导航栏（复用）─────────────────────┐
│                                    │
│  .main (max-width: 720px)          │
│   ┌─ .form-card ────────────────┐  │
│   │  📜 隐私策略                  │  │
│   │  最后更新：2026年6月          │  │
│   │                              │  │
│   │  [政策全文，各节用 h2/h3]      │  │
│   │                              │  │
│   │  联系方式：privacy@...        │  │
│   └──────────────────────────────┘  │
└────────────────────────────────────┘
```

### 2.3 内容（摘自 docs/privacy-policy.md，精简为9节）

1. 数据收集范围
2. 数据使用原则
3. 内容所有权
4. 第三方共享
5. 安全措施
6. Cookie 策略
7. 未成年人保护
8. 策略更新
9. 联系方式

### 2.4 页脚入口

在页面底部（`</body>` 前，不属于 SPA 路由区域）加一行居中文本：

```html
<div style="text-align:center;padding:24px 0;font-size:12px;color:var(--text-sec)">
  © 2026 BrickPlans · <a href="#/privacy" style="color:inherit">隐私策略</a>
</div>
```

**位置**：`index.html` 中 `#app` 容器之下。

---

## 三、后端

无需改动。隐私策略为纯静态内容，无 API 调用。

---

## 四、验收标准

- [ ] 访问 `#/privacy` 能看到完整的隐私策略内容
- [ ] 页面底部有「隐私策略」链接，点击能跳转
- [ ] 模板模式下（无 JS）也能看到页脚链接
- [ ] 内容与 `docs/privacy-policy.md` 一致
- [ ] 样式复用 `.form-card`，统一设计语言
