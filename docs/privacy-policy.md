# BrickPlans 隐私策略 · 产品设计文档

> **定位**：积木图纸分享社区。用户上传作品（图片+元数据），其他用户浏览、点赞、收藏、评论。

---

## 一、数据收集范围

| 数据类型 | 收集项 | 用途 | 是否必需 |
|---------|--------|------|---------|
| 账户信息 | 用户名、邮箱、密码（加密） | 登录鉴权、找回密码 | ✅ 必需 |
| 个人资料 | 头像、简介 | 社区展示 | 可选 |
| 作品内容 | 图片、标题、描述、标签、分类、难度、零件数 | 社区核心功能 | ✅ 必需 |
| 互动数据 | 点赞、收藏、评论 | 社区互动 | 自动记录 |
| 日志数据 | IP、访问时间、User-Agent | 安全审计、反滥用 | 自动记录 |

## 二、数据使用原则

1. **最小化原则**：只收集产品功能必需的数据
2. **透明原则**：用户可随时查看/导出自己的全部数据（「我的资料 → 导出数据」）
3. **可控原则**：用户可随时编辑/删除自己的作品和个人资料
4. **无追踪原则**：不使用第三方追踪脚本（Google Analytics 除外，需声明）
5. **无广告定向**：不做用户画像用于广告投放

## 三、内容所有权

- **用户拥有其上传作品的全部版权**，BrickPlans 仅获得展示权限
- 用户删除作品后，服务器副本在 30 天内彻底清除
- 用户注销账号后，所有个人数据在 30 天内匿名化或删除

## 四、第三方共享

- 不向任何第三方出售/共享用户数据
- 图片存储使用自托管方案（当前为本地存储），不经过第三方 CDN
- 法律要求时（如公安机关调查），依法提供必要数据

## 五、安全措施

- 密码使用加密存储，不存明文
- API 使用 JWT Token 鉴权，30分钟过期 + 刷新机制
- 用户只能编辑/删除自己的内容（后端鉴权校验）
- SQL 使用参数化查询，防止注入

## 六、Cookie 策略

- 登录后本地存储 JWT Token（localStorage），不直接使用 Cookie
- 不使用第三方 Cookie

## 七、未成年人保护

- 注册时不要求年龄，但建议用户在个人资料中标注
- 发现未成年人不当内容可举报，管理后台优先处理

## 八、策略更新

- 修改隐私策略时，通过站内通知 + 邮箱告知
- 重大变更需要用户重新确认同意

## 九、联系方式

- 联系邮箱：`privacy@brickplans.com`（页面底部展示）
- 举报入口：每个作品详情页底部（不起眼的小字链接）
