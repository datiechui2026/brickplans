# PRD-18: 用户设置 & 作品管理

| 字段 | 内容 |
|------|------|
| 功能 | 编辑资料 + 头像 + 我的作品管理 + 密码修改 |
| 优先级 | **P0（产品闭环必须）** |

## 1. 产品目标

用户需要能管理自己的信息和作品。当前完全缺失——这是产品最大的缺口，高于所有运营功能。

## 2. 用户故事

- 作为用户，我点击自己的头像 → 进入个人主页 → 看到「编辑资料」按钮 → 修改头像/昵称/简介
- 作为创作者，我在「我的作品」中看到所有上传 → 可以编辑标题/描述 → 可以删除不需要的
- 作为用户，我可以在「设置」中修改密码

## 4. 后端 API

### 4.1 编辑资料

**`PUT /api/auth/me`**（扩展已有 auth.py）

```json
// Request
{
  "username": "新昵称",
  "bio": "新的个人简介"
}

// Response
{
  "id": "uuid",
  "username": "新昵称",
  "email": "user@email.com",
  "avatar_url": null,
  "bio": "新的个人简介",
  "created_at": "..."
}
```

验证：username 唯一、不与其他用户冲突。

### 4.2 修改密码

**`PUT /api/auth/password`**

```json
// Request
{
  "current_password": "xxx",
  "new_password": "yyy"
}
// Response: 200 OK or 400 当前密码错误
```

### 4.3 头像上传

复用已有的 images API 逻辑，或直接扩展 auth：

**`POST /api/auth/avatar`**
- 接收单个图片文件
- 最大 2MB
- 返回 `avatar_url`
- 自动更新 User.avatar_url

个人主页显示头像时优先用 `avatar_url`，无头像时用首字母圆形。

### 4.4 编辑/删除作品（API已有！）

蓝图 API 已支持 PUT 和 DELETE：
- `PUT /api/blueprints/{id}` — 已存在
- `DELETE /api/blueprints/{id}` — 已存在

只需在前端接入。

## 5. 前端实现

### 5.1 个人主页编辑按钮

```js
// renderUserProfile() 中，如果查看的是自己的主页
if (state.user && state.user.username === profile.username) {
  container.appendChild(
    h('button', { className: 'btn btn-white btn-sm', onclick: () => openSettings() }, '✏️ 编辑资料')
  );
}
```

### 5.2 设置 Modal

- 新建 `openSettings()` 函数
- 2个 tab：个人资料 / 修改密码
- 个人资料表单预填当前值
- 保存时调用 `PUT /api/auth/me`
- 头像上传调用 `POST /api/auth/avatar`

### 5.3 作品管理按钮

Designs tab 中，为卡片加条件渲染的操作按钮：
```js
// 在作品卡片上加
if (state.user && state.user.username === username) {
  // 编辑按钮 → 打开编辑 Modal  
  // 删除按钮 → 确认后调用 DELETE /api/blueprints/{id}
}
```

### 5.4 编辑作品 Modal

- 编辑标题、描述、分类、难度
- 调用 `PUT /api/blueprints/{id}`
- 成功后刷新列表
