# PRD-05: 种子内容脚本

| 字段 | 内容 |
|------|------|
| 功能 | 预置种子数据，让网站上线时有内容可看 |
| 优先级 | P0 |
| 预估 | 0.5天 |

---

## 1. 产品目标

网站上线时不能是空的。需要预置 20-30 条高质量的积木图纸，涵盖各分类，让首批访客有内容可浏览、有参考可看。

## 2. 数据要求

- **数量**：20-30 条
- **分类覆盖**：建筑、车辆、机甲、奇幻、科幻、场景，每个至少 3 条
- **难度覆盖**：1-5 星均有
- **标题/描述**：中文，真实感强，不做作
- **用户**：创建一个官方账号 "BrickPlan官方" 作为作者
- **图片**：先用占位颜色或 emoji（等图片上传功能完成后替换）

## 3. 实现方案

### 新建文件: `backend/seed.py`

```python
# 独立脚本，不依赖 FastAPI
# 使用同步 SQLAlchemy 连接
# 流程：
#   1. 检查是否已有种子数据（幂等）
#   2. 创建官方用户
#   3. 批量插入蓝图
#   4. 输出结果

# 运行方式：cd backend && python seed.py
```

### 种子数据大纲

```python
SEED_DATA = [
    {"title": "中世纪城堡", "category": "建筑", "difficulty": 4, "piece_count": 2500, ...},
```

## 4. 验收标准

- [ ] 运行 `python seed.py` 成功插入 25 条数据
- [ ] 二次运行不重复插入（幂等）
- [ ] 各分类均有作品
- [ ] `/api/blueprints` 列表能查到种子数据
- [ ] 种子数据 `is_published=true`
