# PRD-06: 标签系统前端

| 字段 | 内容 |
|------|------|
| 功能 | 上传时添加标签 + 详情页展示 + 按标签筛选 |
| 优先级 | P0 |
| 预估 | 0.5天 |

---

## 1. 产品目标

标签是内容组织的关键维度。用户上传时可打标签，浏览时可点击标签发现同类作品。

## 2. 交互设计

### 上传页面——标签输入
```
┌─────────────────────────────────────┐
│  标签                               │
│  ┌─────────────────────────────────┐│
│  │ 机甲 科幻  [输入新标签...]     ││
│  └─────────────────────────────────┘│
│  💡 按回车添加标签                  │
└─────────────────────────────────────┘
```

### 详情页——标签展示（可点击）
```
🏷️ 标签: [机甲] [科幻] [大型] [电动]
              ↓ 点击跳转发现页筛选
```

### 发现页——标签筛选区
```
📂 分类: [全部] [建筑] [车辆] ...
🏷️ 热门标签: [机甲] [科幻] [城堡] [赛车] ...
```

## 3. 实现要点

### 后端

已存在 tag 相关模型和 API（开发02AI 提前加了 `tags.router`），确认端点：
- `GET /api/tags` — 获取所有标签
- `POST /api/blueprints/{id}/tags` — 为图纸添加标签

### 前端 (`frontend/src/main.js`)

**上传页面** `renderUpload()`：
- 标签输入框，回车添加
- 已添加标签展示为可删除chip
- 提交时一起发送标签

**详情页** `loadDetail()`：
- 标签区域每个tag可点击 → `navigate('explore', { tag: tagName })`

**发现页** `renderExplore()`：
- 新增标签筛选下拉/按钮组
- 或：热门标签横排展示在分类下方

## 4. 验收标准

- [ ] 上传时能添加多个标签（回车添加、点击删除）
- [ ] 创建图纸时标签同步保存
- [ ] 详情页标签可点击，跳转发现页并筛选
- [ ] 发现页可按标签筛选
- [ ] 标签列表支持搜索/自动补全（可选）
