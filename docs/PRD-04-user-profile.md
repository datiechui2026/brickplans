# PRD-04: 用户个人主页

| 字段 | 内容 |
|------|------|
| 功能 | `/user/{username}` 展示用户信息、作品集、收藏 |
| 优先级 | P0 |
| 预估 | 1天 |

---

## 1. 产品目标

每个用户拥有公开个人主页，展示其：
- 头像、用户名、简介、注册时间
- 上传的作品集（分页）
- 收藏的作品（可选 tab 切换）
- 统计数据（作品数、收藏数）

这是社区感的核心——让创作者有「这是我的主页」的归属感。

## 3. API 设计

### 后端

**新建文件**: `backend/app/api/users.py`

```
GET /api/users/{username}
  返回 UserOut + stats（作品数、收藏数）

GET /api/users/{username}/blueprints?page=&size=
  返回该用户的作品列表（分页）

GET /api/users/{username}/favorites?page=&size=
  返回该用户收藏的作品列表（分页）
```

**修改文件**: `backend/app/main.py`
- `include_router(users.router)`

### 前端

**新增路由**: `#/user/{username}`

**修改文件**: `frontend/src/main.js`
- `pages` 数组新增 `user`
- 新增 `renderUser()` + `loadUserData()`
- 导航栏已登录状态：用户名可点击跳转个人主页
- 详情页作者名、评论头像：可点击跳转个人主页

## 4. 验收标准

- [ ] 访问 `/user/{username}` 展示用户信息和作品列表
- [ ] tab 切换「我的作品」/「我的收藏」
- [ ] 作品卡片点击跳转详情
- [ ] 统计数据正确
- [ ] 不存在的用户返回 404
- [ ] 点击导航栏用户名跳转自己的主页
- [ ] 评论中用户名可点击跳转
