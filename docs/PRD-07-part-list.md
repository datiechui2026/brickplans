# PRD-07: 零件清单展示

| 字段 | 内容 |
|------|------|
| 功能 | 详情页展示零件清单（可视化） |
| 优先级 | P1 |
| 预估 | 0.5天 |

---

## 1. 产品目标

MOC 图纸的核心价值之一就是零件清单。用户在详情页能直观看到所需零件及数量，方便准备材料。

## 3. 实现要点

### 数据格式

当前 `part_list` 字段为 JSON（模型已定义）。约定格式：
```json
{
  "parts": [
    {"name": "2x4 Brick", "count": 45, "color": "红色", "part_id": "3001"},
    {"name": "1x2 Plate", "count": 120, "color": "浅灰", "part_id": "3023"}
  ],
  "total": 1247
}
```

### 上传页面

- 可选：零件清单 JSON 编辑器（textarea）或结构化表单
- MVP：textarea 输入 JSON，前端验证格式

### 详情页

- 零件清单区域，表格展示
- 无零件数据时该区域不显示

### 后端

无需修改（part_list 作为 JSON 已支持）

## 4. 验收标准

- [ ] 有零件清单的图纸详情页展示零件表格
- [ ] 总数统计正确
- [ ] 无零件数据时不显示该区域
- [ ] 上传时能提交零件清单（可选）
