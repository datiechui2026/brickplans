# PRD-02: 评论输入框

| 字段 | 内容 |
|------|------|
| 功能 | 详情页底部评论输入+发表 |
| 优先级 | P0 |
| 预估 | 0.25天 |

---

## 1. 产品目标

用户在图纸详情页浏览评论时，能直接在页面底部输入评论并发表，形成互动闭环。

当前现状：详情页评论区只展示已有评论（只读），用户无法发表。

## 2. 用户故事

- 已登录用户在详情页底部看到评论输入框
- 输入评论内容后点击「发表」，评论立即出现在列表顶部
- 未登录用户看到提示「登录后发表评论」，点击跳转登录弹窗
- 发表成功后输入框清空，无需刷新页面

## 4. 实现要点

### 前端 (`frontend/src/main.js` — `loadDetail()`)

在评论列表下方新增：
- 已登录 → 输入框 + 发表按钮
- 未登录 → 「🔑 登录后发表评论」按钮，点击唤起登录弹窗

发表流程：
1. 按钮点击 → `api.createComment(blueprintId, content)`
2. 成功后 → 清空输入框 → 重新加载评论列表（或乐观插入顶部）
3. 失败 → alert 错误信息

### 后端

无需修改（POST `/api/blueprints/{id}/comments` 已实现）

## 5. 验收标准

- [ ] 已登录用户能看到输入框并成功发表评论
- [ ] 发表后评论立即出现在列表（无需手动刷新）
- [ ] 未登录用户看到登录引导
- [ ] 空内容发表给出提示
- [ ] 发表失败有错误提示
